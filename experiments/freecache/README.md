# FreeCache Exploration Archive

This directory is the small survival kit for the paused FreeCache / TracePlanner exploration.

The experiment is intentionally stopped here. Generated images, JSON/YAML run outputs, policy dumps, copied configs, plots, and GPU vector data were removed. What remains is only the reasoning trail and the smallest reusable CPU-side workflow code for metrics and path search.

## Read Order

1. `STATUS.md` - current milestones, core conclusions, and restart guidance.
2. `METHOD.md` - method-level summary of edge-DP, costs, and validated schedules.
3. `WORKLOG.md` - chronological notes from the exploration.
4. `traceplanner/CONTEXT.md` - details for failed TracePlanner / checkpoint attempts.

## Remaining Core Code

The remaining scripts are not benchmark launchers. They are the core offline metric / workflow pieces:

| File | Purpose |
| --- | --- |
| `tools/order_probe_steptrace.py` | Load steptrace vector payloads and shared JVP helpers. |
| `tools/traceplanner_edge_dp.py` | CPU edge-DP planner over steptrace vectors. |
| `tools/traceplanner_dataset_dp.py` | Dataset-level aggregation of edge costs across prompt/seed traces. |
| `tools/edge_dp_vs_hqphase.py` | Diagnose which edge cost explains a verified ground-truth schedule. |

These scripts expect fresh steptrace vectors generated by the current ChituDiffusion runtime. No old `.pt` vectors are kept here.

## Removed On Purpose

- `archive/` and `artifacts/`: policy JSON, CSV summaries, prompt datasets, and other result data.
- `run_*.sh`: historical launch/config wrappers.
- ChituBench temporary result folders, contact sheets, plots, and closed-loop helper scripts outside this directory.

## If Someone Restarts This

Recreate data from scratch:

1. Run origin / steptrace with the current model adapter.
2. Use `tools/traceplanner_edge_dp.py` or `tools/traceplanner_dataset_dp.py` to propose schedules.
3. Replay schedules through the committed FreeCache runtime path.
4. Treat replay-vs-DP state gap as a calibration signal, not as the direct search objective.
5. Rebuild any plots or reports from new outputs only.

The old exploration is deliberately not reproducible byte-for-byte anymore. It is preserved as conclusions, not as artifacts.
